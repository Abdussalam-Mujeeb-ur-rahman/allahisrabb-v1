API workflows


I draft API workflows for several reasons, which are;


  • To ensure that everyone is on the same page regarding how different components interact and the data flow within the system
  • newcomers can quickly grasp the structure and functioning of the API
  • it helps facilitate productive thinking, and solve problems.
  • without reading the API codes, you already know how it is meant to function.
